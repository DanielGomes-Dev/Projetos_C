Exatamente! Você tocou em um ponto fundamental da manipulação de arquivos em C. A diferença real entre `fgetc`/`fputc` e `fscanf`/`fprintf` está no **nível de abstração** e na **unidade de processamento**.

Embora no fim das contas tudo vire caracteres no arquivo, a forma como o programa "enxerga" esses dados muda completamente.

---

### 1. `fgetc` e `fputc`: O Nível Atômico (Caractere)
Essas funções lidam com a **unidade mínima** de um arquivo de texto, como mencionado no seu material.

* **`fgetc(fp)`**: Lê exatamente **um único caractere** (incluindo espaços, tabs e `\n`).
* **`fputc(c, fp)`**: Escreve exatamente **um único caractere**.

**Vantagem:** Você tem controle total. Se precisar contar quantos espaços existem ou identificar uma quebra de linha específica, elas são ideais.
**Desvantagem:** É trabalhoso para o programador. Se você quer ler a palavra "Casa", teria que chamar `fgetc` quatro vezes e montar a string manualmente em um vetor.

---

### 2. `fscanf` e `fprintf`: O Nível Semântico (Palavra/Formato)
Essas funções são "inteligentes". Elas não olham apenas para letras, mas para **padrões**.

* **`fscanf(fp, "%s", string)`**: Ela faz o trabalho pesado para você. Ela pula os espaços iniciais, lê todos os caracteres até o próximo espaço e já coloca o terminador `\0` no final da string.
* **`fprintf(fp, "%s", string)`**: Escreve a string inteira de uma vez.

**Vantagem:** Produtividade. No seu exercício de retirar palavras repetidas, o `fscanf` é muito melhor porque ele já te entrega a "palavra" pronta para comparar com `strcmp`.
**Desvantagem:** Menos controle sobre caracteres invisíveis (como saber se entre duas palavras havia um espaço ou um TAB).

---

### Comparação Técnica

| Característica | `fgetc` / `fputc` | `fscanf` / `fprintf` |
| :--- | :--- | :--- |
| **Unidade** | Caractere (byte) | Formatada (palavra, número, etc.) |
| **Espaços** | Lê como qualquer outro caractere | Geralmente usa como delimitador (pula) |
| **Facilidade** | Baixa (exige mais lógica do programador) | Alta (faz o parsing automático) |
| **Uso Ideal** | Criptografia, contagem de letras, análise de sintaxe | Processamento de dados, relatórios, leitura de listas |

---

### Por que usamos `fscanf` no seu exercício?

Se usássemos `fgetc`, o código seria muito mais complexo. Teríamos que:
1. Criar um loop para ler letra por letra.
2. Identificar onde começa e termina uma palavra (olhando os espaços).
3. Armazenar essas letras num array temporário.
4. Só então comparar.

Com o **`fscanf`**, o "Fluxo (Stream)" já nos entrega a unidade que nos interessa: a **palavra**.



### O impacto no Cursor
* Com **`fgetc`**, o cursor do arquivo avança **1 byte** por vez.
* Com **`fscanf`**, o cursor "salta" o tamanho da palavra inteira mais os espaços que a cercam.

Ficou clara a diferença de "granularidade" entre elas? No fundo, o `fscanf` é como se fosse um loop de vários `fgetc` empacotados em uma função mais inteligente!
